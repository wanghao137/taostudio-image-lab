# 生图工具使用指南（给普通用户）

TaoStudio 生图平台提供三种生成图片的方式。这份指南用大白话讲清楚：每种是什么、怎么选、怎么用。

---

## 先搞懂：三种方式是什么关系

可以把它想象成"盖房子"的三层：

```
┌─────────────────────────────────┐
│  方式三：独立 Skill（命令行工具） │  ← 给喜欢敲命令行的人
├─────────────────────────────────┤
│  方式二：MCP（给 AI 助手用的接口）│  ← 给 Cursor/Claude 等 AI 用的
├─────────────────────────────────┤
│  方式一：Task API（核心引擎）     │  ← 上面两个都靠它干活
└─────────────────────────────────┘
```

- **方式一 Task API**：是"发动机"。它真正负责生成图片、保证比例不变形、保存文件。一般你不用直接碰它，但上面两种方式都离不开它。
- **方式二 MCP**：是"给 AI 助手插的插座"。如果你用 Cursor、Claude Desktop 这类 AI 工具，让 AI 帮你生图，就走这条。
- **方式三 独立 Skill**：是"命令行小工具"。你在终端敲一行命令就能出图，适合写脚本批量生成、或集成到自己的流程里。

**绝大多数普通用户只需要看【方式一 · 网页版】这一节就够了。**

---

## 方式一 · 网页版（最简单，推荐）

直接打开浏览器访问：**https://image.taostudioai.com/**

### 怎么生成第一张图

1. **打开网站**，在设置里填好你的图片生成 API 配置（地址、密钥、模型名）。
2. **选接口模式**（重要）：
   - 如果你的模型是图像模型（如 `gpt-image-2`、`dall-e-3`）→ 选 **Images API**
   - 如果你的模型是文本模型但能出图（如 `gpt-5.6-sol`）→ 选 **Responses API**
   - 不确定？先用 Images API，这是最常见的。
3. **输入提示词**（想要什么样的图），选好尺寸/比例（比如 16:9 横图、9:16 竖图）。
4. **点生成**，等一会儿就能看到图。

### 两种接口模式的区别

| | Images API | Responses API |
|---|---|---|
| 适用模型 | 专门的图像模型（gpt-image-2 等） | 能出图的文本模型（gpt-5.6-sol 等） |
| 例子 | `gpt-image-2`、`dall-e-3` | `gpt-5.6-sol` |
| 怎么选 | 默认选这个 | 只有当你的模型是文本模型时才选 |

**记住**：模型和接口模式要匹配。图像模型配 Images API，文本模型配 Responses API，选错了会报错。

网页版会自动帮你处理这些：保证图片比例不变形、放大到高清、保存文件。你不用操心技术细节。

## 方式二 · MCP（给 AI 助手用）

如果你用 Cursor、Claude Desktop、或别的支持 MCP 的 AI 工具，可以让 AI 直接帮你生图。

### 它适合谁

- 经常用 AI 编程助手，想"对话式"生图（"帮我生成一张 9:16 的城市夜景"）
- 想让 AI 自动连续生成多张、或把生图嵌进自动化流程

### 怎么配置（给 AI 工具填）

先在本机启动 Task API 引擎（见方式三的启动步骤），然后在你的 AI 工具的 MCP 配置里填：

```json
{
  "command": "node",
  "args": ["D:/你的路径/taostudio-image-lab/server/task-api/mcp-server.mjs"],
  "env": {
    "IMAGE_TASK_API_URL": "http://127.0.0.1:9789",
    "IMAGE_TASK_API_TOKEN": "你在启动服务时设的令牌"
  }
}
```

配好后，AI 助手会获得 6 个能力（你不用记，AI 会自己用）：

| 能力 | 作用 |
|---|---|
| 上传图片 | 把本地 PNG 传上去当参考 |
| 创建任务 | 提交一个生图请求 |
| 查询状态 | 看图生成到哪一步了 |
| 等待完成 | 一直等到出图或失败 |
| 取消任务 | 中途叫停 |
| 下载图片 | 把生成的图存到本地（不会偷偷覆盖已有文件） |

详细的接口参数看 [`docs/image-task-mcp.md`](./image-task-mcp.md)。

## 方式三 · 独立 Skill（命令行工具，已开源）

一个可以单独下载使用的命令行小工具，开源在 [generate-image-asset-skill](https://github.com/wanghao137/generate-image-asset-skill)（MIT 协议，免费用）。

### 它适合谁

- 习惯用命令行的人
- 想写脚本批量生成、定时生成
- 想把生图能力集成进自己的程序

### 快速上手

**第 1 步：装好环境**
- 装 [Node.js](https://nodejs.org/)（18 以上）
- 装 [Python](https://python.org/)（3.10 以上）
- 启动 Task API 引擎（见下面"先启动引擎"）

**第 2 步：准备提示词文件**

把想要的内容写进一个文本文件，比如 `prompt.txt`：
```
一只橘色猫咪在京都町家木缘侧上，午后阳光，写实摄影
```

**第 3 步：敲命令出图**

```bash
# 用图像模型（如 gpt-image-2）——最常见
py -3 scripts/run_image_job.py \
  --backend task-api \
  --api-url http://127.0.0.1:9789 \
  --prompt-file prompt.txt \
  --model gpt-image-2 \
  --api-mode images \
  --provider configured \
  --size 2160x3840 \
  --quality high \
  --out my-image.png
```

```bash
# 用文本模型（如 gpt-5.6-sol）——走 Responses API
py -3 scripts/run_image_job.py \
  --backend task-api \
  --api-url http://127.0.0.1:9789 \
  --prompt-file prompt.txt \
  --model gpt-5.6-sol \
  --api-mode responses \
  --provider configured \
  --size 2880x2880 \
  --quality high \
  --out my-image.png
```

**第 4 步：看结果**

成功后会生成三个文件：
- `my-image.png`：最终的高清大图
- `my-image-source.png`：原始比例的源图
- `my-image.report.json`：生成报告（尺寸、校验信息等）

### 常用参数速查

| 参数 | 作用 | 常用值 |
|---|---|---|
| `--size` | 最终图片尺寸，决定比例 | `2160x3840`(9:16竖)、`3840x2160`(16:9横)、`2880x2880`(1:1方) |
| `--model` | 用哪个模型 | `gpt-image-2`、`gpt-5.6-sol` |
| `--api-mode` | 接口模式 | `images`(图像模型)、`responses`(文本模型) |
| `--quality` | 画质 | `high` |
| `--out` | 输出文件名 | `my-image.png` |
| `--force` | 覆盖已有同名文件 | 不加则不会覆盖（安全） |

> **关键**：`--model` 和 `--api-mode` 必须配套。图像模型用 `images`，文本模型用 `responses`。

详细参数看 [`docs/generate-image-asset-skill.md`](./generate-image-asset-skill.md)。

## 先启动引擎（Task API）

方式二（MCP）和方式三（Skill）都需要先在本地把"发动机"跑起来。网页版不用（网站已经帮你跑了）。

### 一次性配置

在 `taostudio-image-lab` 目录下建一个 `.env.local` 文件（**这个文件不要分享、不要上传，里面有密钥**）：

```dotenv
IMAGE_TASK_API_TOKEN=你自己随便编一串长字符当令牌
IMAGE_TASK_API_PORT=9789
IMAGE_TASK_PROVIDER_BASE_URL=https://你的图片服务地址/v1
IMAGE_TASK_PROVIDER_API_KEY=你的密钥
IMAGE_TASK_PROVIDER_MODEL=gpt-image-2
```

### 启动服务

```bash
npm run task-api
```

看到 `TaoStudio Image Task API listening at http://127.0.0.1:9789` 就说明发动机转起来了。这个窗口要一直开着，关了就不能生图了。

## 常见问题

**Q：生成失败提示"requires an image model"怎么办？**
A：你用文本模型（如 gpt-5.6-sol）却选了 Images API。把模式改成 Responses API 就行。

**Q：图变形/有黑边怎么办？**
A：不会的。系统会强制保证比例——源图和成品图是严格同一个比例，绝不拉伸或补黑边。如果真出现了，那是 bug，请反馈。

**Q：能同时用多个模型吗？**
A：能。同一个引擎可以同时服务图像模型（images 模式）和文本模型（responses 模式），互不干扰。

**Q：重新生成会重复扣费吗？**
A：不会。用同一个 idempotencyKey（幂等键）重试，算同一个任务，不重复计费。

**Q：密钥安全吗？**
A：密钥只存在你本地的 `.env.local`，不会出现在日志、报告、图片里。`.env.local` 已被设置为不上传。
