# 移动端交互设计 — TaoStudio Image Lab

- **状态**：已确认设计（待实现规划）
- **日期**：2026-07-06
- **生产环境**：https://image.taostudioai.com/
- **决策来源**：brainstorming 会话（用户三连确认）
  1. 移动端定位 = 随时随手生图（轻生成、画廊消费）
  2. 实现策略 = 一套代码、响应式重排（不拆移动端组件树）
  3. 输入范式 = 抽屉式输入（从底部滑出全屏专注模式）

---

## 1. 问题诊断（第一性原理）

当前 App 是「桌面三段式 + 固定底部重输入条」硬塞进窄屏，证据来自 390×844（iPhone 14）真实渲染 + 代码结构：

| # | 病灶（证据） | 后果 |
|---|---|---|
| 1 | `InputBar` `fixed bottom-3`，`main` 用 `pb-48`（12rem）让位 | 底部输入条 + 多行参数常驻，吃掉约 40% 屏高 |
| 2 | 输入框下方默认平铺「当前生成策略 / 4K 区块 / 全部参数行」 | 首屏"准备生成一张图"要在窄条里翻很久；主操作（生成）被挤到视觉末梢 |
| 3 | Tailwind 无自定义断点，全是 `sm:` 打补丁；`Header` 里已有 `agentMobileHeaderVisible` 这种 hack | 没有统一的移动布局系统，每加一个特性就再打一块补丁 |
| 4 | `TaskGrid`、`DetailModal`（67KB）、`SettingsModal`（126KB）都是桌面密度 | 模态在手机上要么溢出要么字小到看不清 |
| 5 | 主导航（画廊/智能体）在 Header 里，但手机 Header 还塞了 logo + 4 个图标按钮 | 顶栏拥挤，主导航反而成了次级信息 |

**一句话**：桌面端是「上下分层 + 底部重输入」，手机端需要的是「全屏内容 + 抽屉式输入 + 底部 Tab 导航」——这是两种不同的信息架构，不是同一套布局的缩放。

---

## 2. 信息架构（移动端独立 IA）

移动端 IA 改为 **「底部 Tab 导航 + 全屏内容 + 右下角 FAB + 抽屉式输入」**。

### 2.1 主屏骨架

```
┌─────────────────────────┐  ← 极简顶栏：logo + 主题切换图标
│                         │
│                         │
│   全屏内容区            │  ← 当前 Tab 的内容，几乎占满屏
│   (画廊 / 收藏 / 我的)  │
│                         │
│                         │
│                    (✦)  │  ← 右下角 FAB：「创作」悬浮按钮（主 CTA）
│                         │
├─────────────────────────┤
│  🖼 画廊  ⭐收藏  ✨创作  👤我的 │  ← 底部 Tab Bar（固定，安全区适配）
└─────────────────────────┘
```

### 2.2 创作抽屉（点 FAB「✨创作」或底部「创作」Tab 触发）

从底部滑出全屏抽屉：

```
┌─────────────────────────┐
│ ⌄  收起                 │  ← 顶部抓手 + 收起按钮
│                         │
│ ┌─────────────────────┐ │
│ │ 描述你想生成的图片…  │ │  ← 大输入框（多行，占视觉中心）
│ │ @ 可挂参考图         │ │
│ └─────────────────────┘ │
│                         │
│ [🖼 尺寸 ▾] [⚡质量▾] [+] │  ← 参数 = 一行胶囊按钮，点开 sheet
│  挂图 N 张               │
│                         │
│ ┌─────────────────────┐ │
│ │   ✨ 生成图像        │ │  ← 主按钮，底部安全区上方
│ └─────────────────────┘ │
└─────────────────────────┘
```

### 2.3 设计原则

- **单一焦点**：手机上一次只做一件事。要么在"看图"，要么在"创作"。抽屉式输入让"创作"成为专注模式，关掉抽屉就回到"看图"。
- **拇指可达**：主操作（生成）放底部；导航放底部 Tab；顶栏只放低频图标。
- **参数降级**：手机上默认只露「尺寸 / 质量 / 数量」三个胶囊，其余（格式/透明/审核/精确尺寸/4K）收进「+」更多 sheet。重型功能（Mask 编辑、Agent）在手机上降级为"建议桌面端"提示或二级页面。
- **桌面端零视觉/交互改动**：所有移动 UI 用 `max-sm:` / `sm:` 双前缀隔离，桌面端（≥640px）渲染路径不新增任何视觉或交互分支。唯一对桌面端代码的触碰是 §6.3 的 `useImageComposer` 等价提取重构（行为不变、UI 不动）。

## 3. Tab 与导航细节

### 3.1 底部 Tab Bar（4 格 + 居中 FAB）

| Tab | 内容 | 对应当前代码 |
|---|---|---|
| **画廊** | TaskGrid 全屏单列大图卡片 + 顶部筛选条（搜索/状态/收藏） | `TaskGrid` + `SearchBar` + `GalleryWorkspaceHeader` 的统计 |
| **收藏** | 收藏夹列表 / 收藏图墙 | `FavoriteCollectionsView` |
| **✨ 创作**（FAB） | 触发底部全屏创作抽屉 `MobileComposeSheet` | 复用 `InputBar` 的核心逻辑 |
| **我的** | 设置入口、主题、安装为应用、操作指南、统计、Provider 状态 | `SettingsModal` 的内容拆成分区列表 |

### 3.2 关键取舍

1. **顶栏极简化**：手机顶栏只留 logo（小）+ 主题切换图标。设置/指南/安装入口全部移到「我的」Tab。
2. **「智能体」入口降级**：Agent 不是移动端重点（`AgentWorkspace.tsx` 65KB，重型多图工作流）。在「我的」里留一个入口「智能体工作台」，点进去提示"该功能建议在桌面端使用"，但仍可进入（不硬堵）。现有 `appMode === 'agent'` 分支保留，只是移动端默认不进。
3. **画廊统计卡片**（任务/生成中/已完成/失败/输出/收藏 6 个数字）在手机上从顶部横向滚动条**降级为顶部一行 2-3 个关键数**（输出 / 生成中 / 收藏），其余折进「我的」。
4. **搜索**从当前独立 `SearchBar` 行，改成画廊 Tab 顶部 sticky 搜索条（点开才展开），节省垂直空间。

## 4. 创作抽屉 `MobileComposeSheet` — 参数降级表

原则：**默认只露「想生成什么 + 多大 + 多张」，其余全收起来**。

| 参数 / 功能 | 桌面端（现状） | 移动端创作抽屉 | 处理方式 |
|---|---|---|---|
| **Prompt 输入框**（@ 挂参考图） | 固定底栏主体 | **主区，多行大输入框** | 抽屉视觉中心，@ 挂图能力保留 |
| **生成按钮** | 底栏右侧 | **底部主按钮**（安全区上方） | 抽屉底部固定 |
| **上传图片** | 独立按钮 | **prompt 区下方独立按钮**（主级） | 常驻可见 |
| **尺寸** | auto 选择器 | **胶囊「尺寸 ▾」** | 常驻可见一行 |
| **质量** | auto 选择器 | **胶囊「质量 ▾」** | 常驻可见一行 |
| **数量** | spinbutton | **胶囊「×N」** | 常驻可见一行 |
| **格式**（PNG/JPG） | 选择器 | 折进「+ 更多」sheet | 点「+」展开 |
| **透明背景** | 开关 | 折进「+ 更多」sheet | 同上 |
| **审核** | auto | 折进「+ 更多」sheet | 同上 |
| **精确尺寸** | 开关 + 输入 | 折进「+ 更多」sheet（默认关） | 同上 |
| **4K 区块**（保持比例/改比例/4K 预览） | 整块平铺 | **「4K」小入口**（在尺寸胶囊旁），点开二级 sheet | 降级为二级，且标注"建议桌面端" |
| **当前生成策略展示**（Images API/同步/600s） | 常驻平铺 | **隐藏**，移到生成中 loading 文案 | 不占用创作空间 |
| **Mask 编辑器** | `MaskEditorModal` | **降级**：点"编辑/遮罩"时提示"建议桌面端" | 不做手机版编辑器 |

### 4.1 抽屉布局（自上而下）

```
[抓手 ⌄ 收起]
[ 大 prompt 输入框（多行） ]      ← 视觉中心
[ 📎 上传图片  ][已挂 N 张预览]
[ 尺寸▾  质量▾  ×N▾  +更多 ]     ← 一行胶囊
[          ✨ 生成图像           ]  ← 底部固定
```

「+ 更多」点开是底部二级 sheet（格式/透明/审核/精确尺寸）。4K 单独小入口避免与"随手生图"主流程混淆。

## 5. 画廊 / 详情 / 设置 的移动端适配规则

| 组件 | 现状（桌面） | 移动端规则 |
|---|---|---|
| **`TaskGrid`** 画廊 | 多列网格 | **单列大图卡片**（图占满宽，prompt 摘要 + 状态在图下）。生成中卡片显示进度。 |
| **`TaskCard`** | 紧凑信息 | 字号加大、点击区 ≥44px；长按 = 上下文菜单（复用 `ImageContextMenu`） |
| **`SearchBar` + 筛选** | 独立行 | 画廊 Tab 顶部 sticky 折叠条，点开才展开 |
| **`DetailModal`**（67KB） | 居中模态 | **全屏 sheet**（`inset-0`），图在上、操作按钮区在下，可下滑收起 |
| **`Lightbox`** | 居中预览 | 全屏 + **左右滑切换**多图 |
| **`SettingsModal`**（126KB） | 巨型模态 | **本次：`<sm` 下用全屏 sheet 包裹现状**（`inset-0`、可下滑关闭、底部安全区），内容暂不拆分。**后续独立计划**：拆成「我的」Tab 的分区子页面（API / 外观 / Provider / 高级）。 |
| **`MaskEditorModal`** | 全功能编辑器 | **降级提示**"建议桌面端"，不实现手机版 |
| **`AgentWorkspace`** | 多图工作流 | 入口在「我的」，进入时提示"建议桌面端"，仍可进入 |
| **`HistoryModal`** | 历史列表 | 移到「我的」→「历史」子页面 |

### 5.1 模态通用规则

所有 `fixed inset-0` 模态在 `<sm` 下：
1. 全屏不居中
2. 顶部留抓手可下滑关闭
3. 主操作按钮 sticky 在底部安全区上方

## 6. 断点与技术边界（保证桌面零改动）

### 6.1 断点策略

不加新断点。用 Tailwind 默认 `sm`（640px）为分界线：
- 移动端 UI 用 `max-sm:` 前缀（`max-width:639px`）包裹
- 桌面端 UI 用 `sm:` 前缀（现状不变）
- 桌面端（≥640）的渲染路径一行代码不改

### 6.2 新增技术件

| 件 | 作用 | 位置 |
|---|---|---|
| `useIsMobile()` hook | `matchMedia('(max-width:639px)')`，用于需要 JS 分支处（如渲染 Tab Bar）。**抽取现有 `InputBar.tsx:395` 和 `HelpModal.tsx:13` 的两份本地副本为统一 hook，并替换那两处引用。** | `src/hooks/useIsMobile.ts` |
| `MobileTabBar` | 底部 4 格 + FAB | `src/components/MobileTabBar.tsx` |
| `MobileComposeSheet` | 创作抽屉，复用 `useImageComposer` | `src/components/MobileComposeSheet.tsx` |
| `MobileTopBar` | 极简顶栏 | `src/components/MobileTopBar.tsx` |
| `useImageComposer` hook | **从 `InputBar` 抽出**的 prompt + 参数状态 + 生成调用逻辑 | `src/hooks/useImageComposer.ts` |
| `useMobileSheet` hook | 抽屉上滑/下滑手势 + visualViewport 键盘适配 | `src/hooks/useMobileSheet.ts` |

### 6.3 `InputBar` 重构边界

把 `InputBar.tsx`（127KB）里"prompt 输入 + 参数状态 + 生成调用"抽成 `useImageComposer` hook。桌面 `InputBar` 改成消费这个 hook（行为不变），移动 `MobileComposeSheet` 也消费同一个 hook（参数降级 UI）。

- 业务逻辑单点维护
- 桌面端只是换了数据来源，UI 完全不动
- 这是整个设计里唯一对桌面端有"侵入"的地方，但属于纯提取重构、行为等价

### 6.4 安全区 / 键盘

- `viewport-fit=cover` 已有，补 `env(safe-area-inset-bottom)` 给 Tab Bar 和创作抽屉主按钮
- 虚拟键盘弹出时用 `visualViewport` API 让创作抽屉内的「生成图像」主按钮始终可见（手机上输入时按钮被键盘遮住是常见坑）。键盘适配只影响创作抽屉内部，不影响主屏布局。

### 6.5 PWA 强化（可选，低优先）

已有 manifest，可在「我的」里加一个"添加到主屏幕"引导（复用现有"安装为应用"按钮逻辑）。

## 8. 范围与非目标

### 8.1 本设计覆盖

- 移动端（`<640px`）独立 IA：底部 Tab + FAB + 创作抽屉
- 画廊/收藏/我的三个 Tab 的移动布局
- 创作抽屉的参数降级与「+ 更多」二级 sheet
- 模态（DetailModal/SettingsModal/Lightbox）的移动端全屏 sheet 适配
- 从 `InputBar` 提取 `useImageComposer`（行为等价）

### 8.2 非目标（明确排除）

- **桌面端（≥640px）的任何视觉或交互改动**——一行代码不改
- 移动端 Mask 编辑器实现（降级为提示）
- 移动端 Agent 工作流的完整优化（降级为提示 + 可进入）
- 4K 在移动端的完整平铺 UI（降级为二级 sheet）
- **SettingsModal 拆成「我的」分区子页面** —— 本次仅用全屏 sheet 包裹现状；拆分留作后续独立计划
- 云存储 / R2 / 元数据持久化（P2，独立轨道）
- 任何 provider/proxy/API 行为改动
